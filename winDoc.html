Guía de Recuperación del Entorno

Windows 11 Insider Preview | Reparación de Store, Servicios y Dependencias

Fase 1Diagnóstico y Logs en Tiempo Real

Cuando herramientas como DISM parecen congelarse (ej. al 62.3%), es necesario inspeccionar los logs crudos para confirmar actividad.

Comandos de visualización de Logs

Ejecutar en una terminal paralela mientras se realiza la reparación:

# Ver Log de DISM (El "Gerente" del proceso)
Get-Content C:\Windows\Logs\DISM\dism.log -Wait -Tail 20

# Ver Log de CBS (El "Obrero" - Component Based Servicing)
# Útil cuando DISM parece trabado pero está descargando
Get-Content C:\Windows\Logs\CBS\CBS.log -Wait -Tail 20

Comandos de Reparación de Imagen

# 1. Reparar archivos locales corruptos
sfc /scannow

# 2. Descargar imagen sana desde Windows Update (Requiere Internet)
DISM /Online /Cleanup-Image /RestoreHealth

Fase 2Reactivación de Servicios (El Núcleo)

Problema Crítico: El error 0x80070422 indica servicios deshabilitados. En servicios protegidos como AppXSvc, PowerShell arrojará "Acceso Denegado".

Método A: PowerShell (Intento Estándar)

Útil para servicios no protegidos (Windows Update, BITS).

$servicios = @("wuauserv", "bits", "cryptSvc", "InstallService")

foreach ($s in $servicios) {
    Set-Service -Name $s -StartupType Automatic
    Start-Service -Name $s
}

Método B: Registro de Windows (Solución Definitiva)

Este método fuerza el cambio a nivel de Kernel, ignorando la protección de la interfaz de usuario. Requiere Reinicio.

# Forzar inicio AUTOMÁTICO (Start=2) en servicios bloqueados

# Servicio de Implementación de AppX (Core de Apps Modernas)
reg add "HKLM\SYSTEM\CurrentControlSet\Services\AppXSvc" /v Start /t REG_DWORD /d 2 /f

# Servicio de Licencias de Cliente (Validación de descargas)
reg add "HKLM\SYSTEM\CurrentControlSet\Services\ClipSVC" /v Start /t REG_DWORD /d 2 /f

# Windows Update (Motor de descarga)
reg add "HKLM\SYSTEM\CurrentControlSet\Services\wuauserv" /v Start /t REG_DWORD /d 2 /f

Fase 3Uso de Winget (CLI)

Si la interfaz gráfica de la Microsoft Store falla (pantalla blanca, cierres), Winget es la alternativa oficial robusta que interactúa directamente con el backend.

Tip: Winget no necesita abrir la Store visualmente, pero sí requiere que los servicios de la Fase 2 estén activos.

Comandos Esenciales

1. Buscar una aplicación

winget search "Nombre de la App"
# Ejemplo: winget search WhatsApp

2. Instalar una aplicación (Por ID)

Se recomienda usar el ID (columna izquierda) para evitar ambigüedades.

winget install ID_DEL_PAQUETE
# Ejemplo WhatsApp: winget install 9NKSQGP7F2NH
# Ejemplo Calculadora: winget install Microsoft.WindowsCalculator

3. Actualizar todo el software

winget upgrade --all

4. Reinstalar la propia Store (Si es necesario)

winget install --id Microsoft.WindowsStore --source winget --force

ExtraReset de la Interfaz Gráfica (Store)

Si los servicios funcionan (Winget baja apps) pero la ventana de la Store sigue fallando, ejecuta este comando para registrar el XML de la interfaz nuevamente.

Get-AppxPackage -AllUsers *WindowsStore* | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}
