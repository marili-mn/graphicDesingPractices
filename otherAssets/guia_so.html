// Q&A_ARCHIVE (Respuestas)

01. LIMITACIÓN "RETENCIÓN Y ESPERA"
RESPUESTA: No siempre se conocen todos los recursos necesarios de antemano.

JUSTIFICACIÓN: Exigir todos los recursos al inicio es ineficiente e impracticable en programación moderna debido a la ejecución condicional y dinámica.

03. CIFRADO ASIMÉTRICO (CONFIDENCIALIDAD)
RESPUESTA: La clave pública del receptor.

JUSTIFICACIÓN: Para que sea secreto, se cierra con el candado público del destino, para que solo él (con su llave privada) pueda abrirlo.

04. FUNCIÓN NÚMEROS DE PUERTO
RESPUESTA: Identificar diferentes procesos corriendo dentro de un mismo host.

JUSTIFICACIÓN: La IP llega a la máquina, el Puerto llega a la aplicación específica (Multiplexación).

05. CERTIFICADO DIGITAL
RESPUESTA: Identificar al titular + Contener su clave pública.

JUSTIFICACIÓN: Nunca contiene la clave privada. Es un documento público firmado por una CA.

06. RIESGO ALTO DE DEADLOCK
RESPUESTA: Reutilizable y No Apropiativo.

JUSTIFICACIÓN: Si el recurso no se consume y no se puede quitar a la fuerza, se crea el escenario perfecto para el bloqueo mutuo.

07. FILTRADO DE TRÁFICO
RESPUESTA: Cortafuego (Firewall).

JUSTIFICACIÓN: Es el dispositivo encargado de aplicar reglas de seguridad para bloquear o permitir paquetes.

08. MÁSCARA DE SUBRED
RESPUESTA: Indicar si otra IP pertenece a nuestra red o no.

JUSTIFICACIÓN: Separa la parte de Red de la parte de Host para decidir si enrutar o entregar localmente.

09. RELACIÓN DE CAPAS (IMAGEN)
RESPUESTA: Todas las asociaciones de la imagen son correctas.

Interred -> IP | Aplicación -> Navegadores | TCP -> Ordenado/Conexión | UDP -> Sin conexión/Rápido.
