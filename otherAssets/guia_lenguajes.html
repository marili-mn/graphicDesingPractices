Guía 2do Parcial

C++ Polimorfismo

Para activar el enlace dinámico (dynamic binding):

  • Método base debe ser virtual.
  • Usar punteros/referencias a clase base.

Modificadores

Alcance de variables y métodos:

  • Private: Solo su clase.
  • Protected: Su clase + Hijas.
  • Public: Todo el programa.

PHP Clases

Sintaxis crítica:

  • Herencia: extends.
  • Constructor: __construct (doble guión).
  • Self Reference: $this->variable.

HTTP & Forms

Protocolo y Web:

  • Redirección: Location: url.
  • Métodos Form: GET y POST.
  • Tipos: Texto, Archivos, Números.

Registro de Auditoría (Preguntas Resueltas)

01. La palabra clave private en un atributo indica:
Respuesta: d. Que solo es accesible desde dentro de la clase.
Razón: Encapsulamiento estricto. Ni siquiera las clases hijas pueden tocarlo.
02. En C++, para polimorfismo dinámico, un método debe ser virtual.
Verdadero
Razón: Sin virtual, el compilador usa enlace estático (mira el tipo del puntero, no el objeto real).
03. ¿Cuál afirmación es correcta en POO?
Respuesta: c. Los métodos protegidos son accesibles en las clases hijas.
Razón: Es la definición de protected. (También es cierto que el comportamiento lo dan los métodos).
04. Función correcta para redireccionar en PHP.
Respuesta: b. Ninguna opción es correcta.
Razón: La opción "a" decía Locate:. Lo correcto es header("Location: ...").
05. ¿Qué tipo de datos puede enviarse por un formulario HTML?
Respuesta: Archivos, Números y Textos.
Razón: HTML soporta input text, number y file (multipart). Todo viaja técnicamente serializado, pero soporta esos tipos de datos semánticos.
06. Método para enviar datos desde formulario HTML.
Respuesta: POST (y GET).
Razón: Si es opción única, elige POST. PUT y DELETE no son nativos de form HTML.
07. [IMG] PHP Herencia (Empleado extends Persona con protected).
Verdadero
Razón: La sintaxis es correcta y al ser protected $nombre, la clase hija puede acceder sin error.
08. [IMG] C++ Acceso p.nombre desde main().
Falso
Razón: El atributo no es público. Desde el main (afuera de la clase) no se puede acceder a miembros privados o protegidos.
09. [IMG] PHP "NO es una implementación correcta" (function construct).
Verdadero
Razón: La afirmación es verdadera porque el código ESTÁ MAL. Al constructor le faltan los guiones bajos: __construct.
10. [IMG] C++ Constructor con parámetros y llamada vacía en main.
Falso
Razón: Error de compilación. Al crear un constructor personalizado Persona(string), se borra el constructor por defecto Persona().
